# Test Baselines

This directory stores regression baselines for Layer 4 integration tests.

## Files

| File | Generated By | Description |
|---|---|---|
| `industrial_vietnam_baseline.json` | `tests/julia/test_integration.jl` | Key metrics from Industrial PV+Storage template with Vietnam defaults applied |
| `commercial_api_baseline.json` | `tests/python/test_integration.py` | Key metrics from commercial rooftop PV template via REopt API |

## How Baselines Work

1. **First run:** If a baseline file doesn't exist, the test generates it from the solver/API results and saves it here.
2. **Subsequent runs:** The test compares current results against the saved baseline. Any metric deviating >5% triggers a test failure.
3. **Updating baselines:** Delete the baseline file and re-run the test to regenerate it. This is needed when:
   - Vietnam data files are updated (new tariff, costs, etc.)
   - REopt.jl or the API version changes significantly
   - Scenario inputs are modified

## Baseline Schema

Each baseline JSON contains key metrics as flat key-value pairs:

```json
{
  "pv_size_kw": 22.0,
  "storage_size_kw": 45.91,
  "storage_size_kwh": 114.38,
  "lcc": 1224115.22,
  "npv": -120872.44,
  "initial_capital_costs": 308688.33,
  "initial_capital_costs_after_incentives": 308688.33,
  "pv_year_one_energy_kwh": 29988.0
}
```

Note: With Vietnam defaults applied (all US incentives zeroed), `initial_capital_costs` should equal `initial_capital_costs_after_incentives`.
