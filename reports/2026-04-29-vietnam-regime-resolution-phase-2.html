03 / Math / Algorithm Used

Math / Algorithm Used

  • Recursive overlay: regime resolution uses deep dictionary merge so only changed policy fragments override the baseline payload. Unspecified fields continue to inherit current repo behavior.
  • Tariff synthesis: the resolved tariff payload still maps TOU windows and multiplier blocks into the existing 8760 hourly series and monthly demand-rate vector.
  • Export cap selection: max_export_fraction now defaults from the resolved export-rule payload, unless a caller explicitly overrides it.
  • Parity gate: identical scenario inputs are processed in Python and Julia for four named regimes and compared recursively with tolerance 1e-10.

06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • The regime system now changes preprocessing outputs, but it still does not run multi-regime matrix execution or delta reporting. Those are Phase 3 and Phase 4 surfaces.
  • The repriced-multiplier Decision 963 bundle remains mostly a structural placeholder because no published replacement multiplier table has been encoded yet.
  • Hourly discretization still approximates half-hour tariff boundaries, so Decision 963 timing deltas remain directional rather than invoice-accurate.
  • Second-best alternative rejected this phase: separate code branches for each regime. The data-driven overlay was better because it keeps both languages aligned and makes future registry updates cheaper.
07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • Expected warning: the 50% rooftop export draft regime emits the existing warning that custom export fractions are metadata for Vietnam solve wrappers, not plain REopt.run_reopt(...) enforcement.
  • Environment warning: Julia emitted the same non-blocking ArchGDAL precompile overwrite warning already seen in this environment, but the unit lane still passed 168 / 168.
  • No parity drift was found across the four tested regimes.
