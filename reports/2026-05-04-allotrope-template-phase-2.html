03 / Math / Algorithm Used

Conformance Scoring Formula

Each of the 12 rubric dimensions (D01–D12) is scored 0–5 via slide-by-slide inspection:

Score Meaning
5Exact match — indistinguishable from template
4Close match — minor deviation, immediately fixable
3Acceptable — conforms in spirit, visible difference
2Partial — some elements present, key aspects missing
1Poor — attempted but largely non-conforming
0Absent — not attempted or completely wrong

Aggregate score = Σ(D01–D12), max 60.   Per-dimension score = median across all applicable slides in the deck.

06 / Limitations / Second-Best Alternative

Limitations & Second-Best Alternatives

  • No logo images available. The Allotrope logo was not bundled as a distributable image file. Deck 1 uses a text placeholder. Second-best: source a low-res logo from the Google Slides template export and embed it directly.
  • pptxgenjs has limited rendering fidelity. It does not support slide masters or theme inheritance from a reference .pptx. Second-best: use python-pptx to post-process the deck after generation for deterministic fixes (colors, fonts, footers).
  • python-pptx inspection is XML-level, not visual. It reads OOXML properties but cannot detect visual issues like text overflow, clipping, or anti-aliasing differences. These require manual visual review.
  • No slide numbers in pptxgenjs output. The library lacks a built-in slide number feature. Second-best: add slide numbers via python-pptx post-processing script.
