4. Math / Algorithm Used

No numerical algorithm was introduced in this phase. The main logic was schema-based:

  • Require `regimes` as a sixth manifest key alongside the existing five Vietnam data surfaces.
  • Enforce complete 24-hour TOU coverage for any regime that overrides `tou_schedule`.
  • Enforce `0 <= max_export_fraction <= 1` for any regime that overrides rooftop export caps.
  • Preserve current behavior by representing the existing repo baseline as `decision_14_2025_current`.

7. Limitations / Second-Best Alternative

Current limitation

Phase 1 only adds the registry surface. No `regime_id` resolution or scenario execution changes exist yet, so the new data is not consumed by preprocessing behavior.

Second-best alternative

Add only a `tou_regime` switch to the tariff builder and defer a full registry. That would be smaller short-term, but it would immediately underfit the broader Idea 2 scope across export caps and demand-charge regimes.
