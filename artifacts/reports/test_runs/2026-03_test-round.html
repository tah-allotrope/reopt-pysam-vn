Root Cause The full optimization payload submitted to the REopt API contains a field or structure that the API rejects. This is not related to the nrel.gov → nlr.gov domain migration — connectivity is confirmed healthy via test_nlr_domain_connectivity (HTTP 200, valid load profile returned).
Regression? No. Both tests failed before commit 081fa45. The reorganisation did not introduce this failure.

Workaround Use the local Julia solver (run_vietnam_scenario.jl) or run test_integration.jl directly for full solve validation while API payload is debugged.
Fix Command 
python -m pytest tests/python/test_integration.py::TestAPIIntegration::test_commercial_rooftop_api_solve -v -s
🟡 NOT RUN tests/julia/test_integration.jl — Julia cold-start timeout ▶
Symptom Julia process does not complete within 5–10 minutes when run via test runner or external tool; appears to hang but is actually precompiling REopt.jl + ArchGDAL + JuMP.
Root Cause First-run Julia precompilation (no sysimage) for a package of REopt.jl's size takes 3–8 minutes on a cold machine. The --compile=min flag reduces JIT but does not bypass sysimage loading. Subsequent runs reuse the depot cache and are much faster.
Not a Bug The test code itself is correct — Python equivalents (same logic, no solver) completed in ~10s. This is a Julia startup overhead issue only.
Fix Run directly in a persistent Julia session, or use the updated test runner with a higher timeout: 
# Option A: Direct (interactive Julia session — avoids cold-start on repeat runs)
$env:JULIA_PKG_PRECOMPILE_AUTO = "0"
julia --project --compile=min tests/julia/test_integration.jl --smoke-only

# Option B: Via test runner with timeout
.\tests\run_all_tests.ps1 -SmokeOnly -JuliaTimeoutSeconds 1200

# Option C: Full solve (first run ~30-60 min, subsequent ~5-10 min)
.\tests\run_all_tests.ps1 -Layer 4 -JuliaTimeoutSeconds 3600

Changes Made This Session

📄
Created scripts/julia/run_vietnam_scenario.jl
New end-to-end run script for the commercial rooftop PV template. Loads NREL API keys from NREL_API.env, applies Vietnam preprocessing defaults, constructs Scenario(), and optionally runs HiGHS solver. Supports --no-solve flag for fast input validation. Fixes the broken Quick Start reference that existed in README.md since commit 081fa45.
✏️
Fixed README.md Quick Start block
Replaced the single broken julia … run_vietnam_scenario.jl line (script did not exist) with three clear steps: set API keys → fast validation via --no-solve → full optimization solve. Updated API key comment to reference NREL_API.env.example.
⚙️
Added -JuliaTimeoutSeconds to tests/run_all_tests.ps1
The test runner's Invoke-Julia function previously used Start-Process -Wait with no timeout, causing indefinite hangs on Julia cold-start. Now accepts -JuliaTimeoutSeconds <int> (default 0 = no limit). When exceeded, the process is killed and a clear TIMEOUT message is printed with a tip to increase the limit. .SYNOPSIS updated with recommended values per scope.
📖
Updated docs/pitfalls.md
Added "L4 Julia Tests — Cold-Start Timeout" section explaining the precompilation overhead, the new -JuliaTimeoutSeconds flag, and a table of recommended limits per test scope (L1/L2: 600s, L4 smoke: 1200s, L4 full solve: 3600s).
📋
Updated AGENTS.md test status table
Split L4 table into separate Python and Julia rows. Added "exact match, max diff 0.00e+00" note for L3. Updated Julia L4 row to document the cold-start issue and the -JuliaTimeoutSeconds 1800 suggestion.
