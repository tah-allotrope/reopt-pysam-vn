Bug Fixes Applied (Phase 2 → 3)

Bug 1 — equity_irr.py: EBITDA used discounted NPV instead of nominal savings  Fixed 2026-03-22
Root cause: extract_annual_ebitda() computed lcc_bau − lcc_opt (both present-value lifecycle costs), producing a severely understated year-1 CF of ~$319K instead of ~$5.93M. This drove equity IRR to −17.9%.

Fix: Year-1 base now reads Financial.year_one_total_operating_cost_savings_before_tax ($5,929,979) and escalates at 5%/yr.

Result: Equity IRR recovered to 19.8% vs. Excel target 19.4% (+0.4 pp).
Bug 2 — compare_reopt_vs_excel.py: energy-flow keys did not match REopt output schema  Fixed 2026-03-22
Root cause: Script used stale key names (year_one_to_load_kwh, year_one_to_grid_kwh, etc.) absent from the actual results JSON.

Fix: Mapping updated to live REopt schema keys: PV production → PV.year_one_energy_produced_kwh; PV export → PV.annual_energy_exported_kwh; grid → ElectricUtility.annual_energy_supplied_kwh; revenue → Financial.year_one_total_operating_cost_savings_before_tax.

Regression coverage added in tests/python/test_saigon18_compare.py.
